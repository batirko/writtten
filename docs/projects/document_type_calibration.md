# Document-type calibration

> Written 2026-07-02 from a live essay-writing session (see `docs/logs/prompt_quality_observations.md` OBS-028 and the pre-existing OBS-023). The product's stated scope broadened the same day from "PRDs" to **documents people write for work, PRD-primary** (`docs/concept.md` § First persona and use case). This doc is the mechanism that makes that breadth safe: it keeps the fixed taxonomy and the lean posture while letting the eval's _strictness_ track the genre.

## Status

The Phase-8 slice now has build-ready designs (§ _Audience-relative jargon calibration_, § _Extraction & tension calibration for rhetoric_): the inferred-audience jargon fix (OBS-003/OBS-005, V1-quantified), the OBS-037 rhetoric levers, with the applicability matrix + per-class floors staying evidence-gated on V1/V2 and the presets idea superseded by the owner's inferred-audience decision.

> **Live gap found 2026-07-08 (OBS-036) — cold-open levers resolved 2026-07-09; richer early-inference remains Phase 8.**
>
> **The floor was inert until the class is known, and on a cold essay with no stage set it never became known in time.** A personal essay evaluated with `Stage/Context: (none set)` classified as `unknown` → the calibration block is gated empty for `unknown` → full PRD strictness applied, and the user dismissed two of the resulting cards (an `unsupported_claim` on a rhetorical apprehension, a `missing_topic` demanding "examples or evidence" of the essay's thesis). Inference only emitted a `suggested_stage` at the terminal `doc-idle`, *after* the flagging. Two follow-ups this exposes: **(1)** the class must be inferred (or the `unknown` default softened) *before* the first strict pass — otherwise every un-staged non-PRD gets PRD treatment on the way in (ties `onboarding_first_run.md`; a Phase-8 "richer inference" item, but the un-staged-default decision could land in Phase 6); **(2)** the always-on first-person carve-out missed a *rhetorical-question* framing ("won't my … muscles atrophy?") — the shipped bullet only lists declarative "I fear…/I'm concerned…" forms; extend it to interrogative/hypothetical framings (small Phase-6 prompt tweak). **No new observation types are warranted** — this stays calibration of existing checks (invariant #2).

The trigger: the eval applies PRD-grade citation strictness regardless of what the document is. A first-person apprehension in an essay ("I fear my writing skill will atrophy") was flagged as an unsupported factual claim (OBS-028); a public-communication narrative device was flagged the same way despite `Document context: a public communication about a product` being in the prompt (OBS-023). The `Document Context / Stage` field already exists and is already passed to the model — but it is treated as flavour text, not as a calibration input. This doc makes it load-bearing.

## Design

### Why calibration, not new types

Broadening from PRDs to "documents people write for work" tempts a genre-specific taxonomy (blog checks, memo checks). That breaks invariant #2 and explodes the surface. The observed failures aren't missing types — the core checks (contradiction, clarity, unsupported claim, missing/underexposed topic) generalise fine. The failures are **strictness applied uniformly**: a citation bar and a section-completeness bar tuned for PRDs, fired on an essay. So the lever is a **calibration input**, not new observation types. The taxonomy stays fixed; each check reads the document class and adjusts how hard it presses.

### Document classes (draft — settle during build)

A small, coarse enum — resist granularity. A working starting set:

- **`prd_spec`** — PRDs, specs, decision docs. The anchor case; full strictness (citation, structure, metrics, scope).
- **`comms_announcement`** — stakeholder comms, announcements, product/feature blog posts, external narrative. Narrative devices and rhetorical framing are expected; `unsupported_claim` only bites hard external-fact assertions; `missing_topic` structural expectations largely off.
- **`memo_email`** — substantial work emails / memos to colleagues. Expect an ask/decision and its rationale; lighter structural expectations than a PRD, heavier than a blog.
- **`essay_personal`** — first-person reflective/opinion writing. Opinions and apprehensions are the content, not claims to cite; contradiction and clarity still apply, `unsupported_claim`/`missing_topic` heavily relaxed.
- **`unknown`** — the conservative default before inference/selection: behave close to today (PRD-ish) but with the OBS-028 opinion carve-out always on.

### What each class dials

Calibration modulates three knobs, not the type list:

1. **`unsupported_claim` strictness** — from "cite any world-fact assertion" (`prd_spec`) to "only hard external-fact assertions, never first-person opinion/narrative" (`essay_personal`).
2. **`missing_topic` / structural expectations** — the expected-section notion (objective, scope, metrics, timeline, risks) is a PRD construct; suppressed or reframed for other classes.
3. **Baseline hedging/register** — off-genre, structural gaps skew further toward soft "opportunities" (composes with `maturity_aware_severity` R2 and `emotional_register`).

Checks that generalise unchanged — **contradiction** (the hero), **clarity**, **undefined_jargon** — stay on across all classes (jargon presets aside).

### Relationship to the stage/context chip

No new control surface. The `Document Context / Stage` value the context chip already exposes (`onboarding_first_run.md` § The context chip) carries the class. Inference sets it; the user can correct it inline. Calibration is a pure consumer of that value — which is exactly why OBS-023 is damning: the value was present and ignored. This doc closes that gap.

### Audience-relative jargon calibration (Phase 8 — design settled 2026-07-16)

**The evidence:** V1 Run 1 measured `undefined_jargon` firing **21–53× per real technical PRD** (vs ~2 on public proxies) — the dominant felt-noise on real documents — with the flags eyeballed as largely audience-inappropriate: domain vocabulary the doc's own intended reader shares (OBS-003 payments terms, OBS-005 product-process terms, now quantified at scale). **Owner decision 2026-07-16: _inferred_ audience, not a declared control and not a bigger allow-list.**

**Mechanism — three composing layers:**

1. **Audience-relative judgment, in-check (prompt).** Rewrite `MERGED_SYSTEM_PROMPT` item 5: a term is flagged only when the **document's intended audience** likely wouldn't know it — the audience being the stated document context when present, else **inferred from the document's own register and vocabulary in the same call** (a doc written in a domain's vocabulary is written for readers who share it). This keeps the zero-config posture — no new chip, no plumbing of unaccepted stage suggestions — and the inference rides the existing fast call at zero extra requests. It preserves the true positive: a term the *inferred* reader plausibly doesn't know ("should you explain this?") still fires. Also instruct: flag each distinct term **once**, choosing its first occurrence, and only the few terms most likely to actually block that reader.
2. **Deterministic once-per-term dedup (client).** The model instruction is soft; the guarantee is code: in `evaluator.ts`'s jargon emit arm, drop a new `undefined_jargon` observation whose normalized term (its `anchorText`) matches any **active** jargon observation elsewhere in the document (`allActiveObs` is already loaded at that point — no new reads). First card wins; an edit that removes the first card lets a later occurrence surface naturally on its section's next eval.
3. **Ranked per-section cap (client).** `JARGON_SECTION_CAP = 3` per settle after dedup, keeping document order (the model already leads with what it judges most blocking; deterministic tie-break by position). Volume above the cap is exactly the "wall of flags" V1 measured — dropping it is the point, not a loss. Composes with (does not replace) the feed's priority budget and the Phase-9 noisiness dial (R2c).

**Hash/fixture impact (accepted):** item 5 is in the static system prompt, so every fast-call hash shifts — an offline byte-identical **re-key** for fixtures whose responses can't change (the `reqHash` re-key procedure from the OBS-027/OBS-036 precedents; note `reqHash` joins `json-flag \n system \n␠\n user` — a newline-space-newline separator, not a NUL byte, `model/mock.ts`), plus a **re-record** of `jargon-allowlist` (its expected behavior genuinely changes: the domain-term FP should now vanish audience-relatively, not via the list). New discrimination fixture `jargon-audience-inferred`: a technical PRD slice where standard domain vocabulary is **not** flagged while a genuinely undefined invented acronym **still is** (rides the shared Phase-8 recording session).

**Acceptance (the V1 tie-back) — result 2026-07-17.** Ran 4 V1 real-PRD docs live (paid key, fast-call jargon check), before (`main`) vs after (this branch), counting `undefined_jargon` cards and distinct terms per doc:

| doc | before cards / distinct | after cards / distinct |
| --- | --- | --- |
| vi-01 | 16 / 9 | 15 / **7** |
| vi-03 | 11 / 10 | 11 / **10** |
| prd-trading-alerts | 18 / 17 | 14 / **14** (11 with a stated eng-audience) |
| study-n8n | 25 / 21 | 20 / **20** |

The deterministic layers deliver their guarantee — **one card per distinct term** (the duplicate wall is gone) and ~20–40% fewer cards — but the **< 5 distinct-term target was not reached**. Root cause: the *fast-tier* model (`gemini-3.1-flash-lite`, which the jargon check runs on) keeps flagging audience-known domain terms/acronyms (`rsi`, `macd`, `oauth`, `websocket`, `gdpr`, `n8n`, `rag`, `nlp`, `llm`) that the audience-relative *prompt* can't suppress at that capability; a stated audience helps only marginally (14→11). **Owner decision 2026-07-17: ship the deterministic layers + improved prompt as-is and re-scope the bar to "once-per-term, per-section-bounded, materially fewer cards."** The `< 5` figure is a **strong-tier property**; the residual (fast-tier acronym over-flagging) folds into the Phase-8 free-tier-expectations milestone — same root cause as the free-tier under-adjudication. A doc-wide ranked cap (would force `< 5`) and a bigger allow-list were both considered and declined: dropping a genuinely-unknown term to satisfy a count silences a live "should you explain this?" provocation (invariant #1's spirit).

### Extraction & tension calibration for rhetoric (OBS-037, Phase 8 — design settled 2026-07-16)

**Lever 1 — extraction-side (the upstream fix).** Under the three relaxed classes **and** `unknown`, `sectionCalibrationBlock` gains an extraction rule alongside the existing `unsupported_claim` rule: *"Do not extract rhetorical or hyperbolic emphasis, narrative color, or evaluative flourishes ('X was a HUGE thing', 'the tipping point for the revolution') as claims of any kind — extract only assertions the author is genuinely committing to, constrained by, or would cite."* Once rhetoric stays out of the ledger, both downstream cascades (false `unsupported_claim`, false `strategic_tension` regenerating per rerun — the OBS-037 paid-churn loop) lose their fuel. `prd_spec` stays untouched (hash-stable strict anchor); the block edit re-keys the unstaged/relaxed fixtures **byte-identically** (responses unchanged — the standard offline re-key).

**Lever 2 — tension-side (the composing guard).** The four contradiction prompts' NOT-A-CONFLICT bucket currently covers only *compatible-but-unspecified* pairs (OBS-031). Add the **author-reconciled** case: *"If the document itself acknowledges the tradeoff and states its resolution (A, but B, so we chose C — a problem statement followed by the change that addresses it), it is a resolved tradeoff, not a live tension or contradiction — report it in neither bucket."* V1 Run 1 measured this exact FP class on real PRDs ("a current-state problem flagged against the proposed fix that resolves it"), so the rule earns its place on evidence, not just the essay case. Strong-call prompt edit → strong-side fixture re-key (responses unchanged for the existing fixtures; the sweep-fidelity fixture asserts messages, not this bucket).

**Verification:** extraction discrimination fixture `rhetoric-extraction` (essay-class section: "Chat interface was a HUGE thing" extracts **zero** claims; "handwriting improves memory retention by 40%" still extracts + flags); a reconciled-tension fixture (an A-but-B-so-C narrative produces **no** tension; a genuinely live tradeoff still does) — both live-recorded in the shared Phase-8 recording session. Note: **the OBS-036-residual carve-out facet (interrogative/hypothetical first-person concern) is already shipped** — #162, 2026-07-09, `rhetorical-apprehension` fixture — the Phase-8 plan milestone narrows to the two levers above.

### Out of scope

- New observation types (invariant #2). Genre needs are met by calibrating existing checks.
- Per-class UI beyond the existing context chip.
- Binary-format import / rich content types (`canvas_content_types.md`) — orthogonal.
